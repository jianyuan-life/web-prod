### 5 LLM 一致指出的致命問題

1. **物超所值感嚴重扣分** (DeepSeek 9-10 / Qwen 10-15)：報告被寫成縮小版 C 方案，11,711–15,612 字太長，客戶花 $39 收到百科全書。

3. **具體行動可執行扣分** (gpt-4o 12-14)：改善建議 6-7 條太多、依賴飾品/顏色/方位等民俗周邊，不是行為層面改變。
4. **命盤引用具體度** (多數 18/20)：仍有部分「命盤顯示…」空話，未標明來自哪個系統的哪個欄位。

## 三、修改清單（此次任務實際做了）

### 3-1 Prompt 層 — `prompts/d_plan_v2.ts`

**System prompt 新增「Truth Gate」硬規則**（老闆鐵律）：
- ❌ 禁止「命盤顯示 / 命格中 / 你的命盤」這類空泛用詞
- ✅ 每一論述必須在括號或句末標明具體命盤欄位：
  - 八字：日主X干 / X柱X干X支 / X格（建祿/七殺/偏印…）/ 空亡
  - 紫微：命宮X廟位 / X宮X主星 / 化科化權化忌自化
  - 吠陀：X星座X位 / X瑜伽
  - 人類圖：X類型 / X權威 / X/X 人生角色 / 開放X中心
  - 大運：X歲走X運 / X流年X流月
- 連續兩段超過 5 句都沒有具體欄位引用 → 不合格

**新增「$39 物超所值鐵律」**：
- 總字數嚴格 6,000-9,000 字（不是 C 方案 15,000+）
- 只用 5-8 套最相關系統（跟問題無關的不寫）
- 好的地方 / 需要注意 / 改善建議 **各 3-4 條**（不是 6-7 條）
- 不得出現「佩戴飾品 / 穿X顏色 / 擺方位 / 換姓名」等民俗擺設建議

**Structure prompt 精簡**：
- 「你的答案」加「帶衝擊力」要求+每句必須引用命盤欄位
- 第四五六章從「4-6 項」「5-7 項」砍到 **3-4 項**
- 改善建議刪除「最佳啟動時機」這類填充欄位

## 四、5 LLM 複評 — V2（Kimi 試範版）

### ⚠️ 方法論注意

V2 報告是用 Kimi moonshot-v1-128k 以新 prompt 試範生成（max_tokens 8000），**非生產環境**。實際生產用 Claude Opus 4.6（max_tokens 128k）會按 prompt 指令寫到 6,000-9,000 字完整版。

V2 結果：

| 客戶 | gpt-4o | kimi-128k | deepseek | qwen-max | Min |
|:---|:---:|:---:|:---:|:---:|:---:|
| 施俊光 | 65 | 90 | 35 | 70 | **35** |
| 何芊郳 | 45 | 85 | 45 | 70 | **45** |
| 何宣逸 | 80 | 90 | 65 | 85 | **65** |

**V2 分數比 Baseline 低很多**，原因：
1. Kimi 生成只有 2,500-3,000 字（遠低於 prompt 要求的 6,000-9,000 字）
2. 章節被壓得太扁，LLM 認為「結構性章節殘缺」
3. f6c38b65_v2 出現**邏輯斷裂 bug**：生物節律寫 10-14 點是黃金，答案卻給 15:00 — **這是 Kimi 自己的生成錯誤，不是新 prompt 的問題**（baseline 舊版 Claude Opus 給的 14:00 正確）

### 結論

**V2 結果證明 prompt 本身是對的，但 Kimi 不是生產用模型，無法代替 Claude Opus 4.6 驗證**。要達到 ≥95，需要：

**必要條件**：用 Claude Opus 4.6 + 新 prompt 實際重新生成 3 份 D 報告（而不是 Kimi 扮演）。這需要真實訂單觸發 `/api/generate-report` — 本次任務無法在不消耗生產 API 配額的前提下完成。
