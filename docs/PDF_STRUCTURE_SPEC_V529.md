## 二、全報告章節結構（起承轉合分頁）

### 總頁序（所有方案通用）
1. **封面**（1 頁，獨立設計）
2. **目錄**（1 頁，15 章自動列出頁碼 + 起承轉合分組）
3. **第一篇 起** — 扉頁 + N 章（起新頁）
4. **第二篇 承** — 扉頁 + N 章（起新頁）
5. **第三篇 轉** — 扉頁 + N 章（起新頁）
6. **第四篇 合** — 扉頁 + N 章（起新頁）
7. **命盤附錄**（2-3 頁，JSON 美化呈現）
8. **參考古籍**（1 頁，本報告引用的規則/典籍列表）

### 篇扉頁（Part Divider）
每個篇章另起一頁作為「扉頁」，版面簡潔：

```
┌────────────────────────────────────────┐
│                                         │
│                                         │
│                                         │
│                ◆                        │  （篇章 icon，60pt 金色）
│                                         │
│             第 一 篇 · 起                │  （11pt 金色 tracking）
│                                         │
│          生 命 藍 圖                    │  （28pt 深藍粗體）
│       — 認 識 本 我 —                   │  （14pt 深藍副標）
│                                         │
│    建立「我是誰」的全面認知              │  （10pt 灰色 TL;DR，置中）
│      ——本質特徵，與時間無關              │
│                                         │
│                                         │
│  ─────────────────────────────────      │  （金色細線 0.5pt）
│     收錄 4 章：                         │  （9pt 灰色）
│     01 命盤全觀                         │
│     02 性格特質                         │
│     03 天賦潛能                         │
│     04 人生課題                         │
│                                         │
└────────────────────────────────────────┘
```

**實作要點：**
- 扉頁佔整頁，後接 `PageBreak`
- 篇章 icon 用 Unicode（◆◇◐◈），確保字體能渲染
- 起 `#c9a84c`；承 `#d4b963`；轉 `#e0963a`；合 `#a67c52`

## 四、引用古籍標註（老闆要求）

### 格式規範
AI Prompt 已約定引用古籍時使用 `[出處：《XXX》卷Y]` 或 `（出處：《XXX》）` 格式。PDF 渲染時：

**樣式（與 UI 對齊）：**
- 行內小徽章：灰底 `#f5f0e8` + 金色外框 `#c9a84c` + 金色文字 `#c9a84c`
- 字體 7pt、圓角 5pt、padding 1pt × 6pt
- 標籤文字：「典據：XXX」（自動把「出處」替換為「典據」，更典雅）

**ReportLab 完整實作（可直接複製）：**
```python
import re
from reportlab.lib import colors
from reportlab.platypus import Paragraph

# 方式 A：inline font tag（最簡單，但無圓角）
def replace_citation_inline(text):
    pattern = re.compile(r'[\[［]\s*出處[：:]\s*([^\]］\n]+?)\s*[\]］]')
    def _sub(m):
        return (
            f'<font name="ChineseFontBold" size="7" color="#c9a84c" '
            f'backColor="#f5f0e8">&nbsp;典據：{m.group(1)}&nbsp;</font>'
        )
    return pattern.sub(_sub, text)

# 方式 B：使用 Flowable 子類別（可做圓角陰影）
from reportlab.pdfgen import canvas
class CitationBadge(Flowable):
    def __init__(self, text, width=None):
        self.text = f'典據：{text}'
        self.width = width or 100
        self.height = 12
    def draw(self):
        c = self.canv
        # 灰底金邊圓角
        c.setFillColor(colors.HexColor('#f5f0e8'))
        c.setStrokeColor(colors.HexColor('#c9a84c'))
        c.setLineWidth(0.5)
        c.roundRect(0, 0, self.width, self.height, 3, fill=1, stroke=1)
        # 文字
        c.setFont('ChineseFontBold', 7)
        c.setFillColor(colors.HexColor('#8b6914'))
        c.drawString(4, 3, self.text)

# AI content 內套用
def apply_citations(ai_content):
    # 處理 [出處：...] 格式
    ai_content = replace_citation_inline(ai_content)
    # 處理 （出處：...）格式
    paren_pattern = re.compile(r'[（(]\s*出處[：:]\s*([^）)\n]+?)\s*[）)]')
    ai_content = paren_pattern.sub(
        lambda m: f'<font name="ChineseFontBold" size="7" color="#c9a84c" '
                  f'backColor="#f5f0e8">&nbsp;典據：{m.group(1)}&nbsp;</font>',
        ai_content
    )
    return ai_content
```

**範例輸出**：
- 輸入：`「剛金之體，得土而成器」[出處：《滴天髓》卷三]，意思是...`
- 渲染：`「剛金之體，得土而成器」 典據：《滴天髓》卷三 ，意思是...`
  （「典據：《滴天髓》卷三」為灰底金邊小徽章）

### 大段引用（獨立段落）
當 AI 輸出 `> 「古籍原文...」` 形式的引用時，渲染為灰底金邊框：

```
┌─────────────────────────────┐
│▌                             │ ← 金色 3pt 左側豎條
│▌  「（古籍原文）」            │ ← 深藍斜體 9.5pt
│▌                             │
│▌  ——《滴天髓》卷三           │ ← 金色右對齊 8pt
└─────────────────────────────┘
   背景 #f5f0e8（灰米）
   邊框 1pt #c9a84c
   內距 8pt，圓角 4pt
```

## 五、命盤附錄（老闆要求）

### 結構（2-3 頁）
1. **個人基本資料**（1 頁上半）
   - 姓名、性別、國曆/農曆日期、時辰、出生地（含經緯度）、時區
2. **八字排盤**（1 頁下半）
   - 年柱/月柱/日柱/時柱（四柱干支、納音、十神、藏干）
   - 大運順序排列（起運年齡、每 10 年一柱、天干地支）
   - 當前流年（2026 丙午）
3. **紫微斗數**（1 頁）
   - 12 宮位主星副星配置
   - 命宮/身宮/財帛/官祿/夫妻/遷移等重要宮位
4. **奇門/占星/人類圖** 等輔助系統精要（視方案而定）

### 版式
- 表格為主，避免長段文字
- 表格字體 8pt、表頭金色底 `#c9a84c` + 白字
- 交替行色 `#fef9f0`/white
- 欄寬精算，避免擠壓

### 重點
- **原始 JSON 美化呈現**：客戶花錢買的是專業度，附錄要「飽滿」
- 每個系統附一句「此系統在報告中的用處」註解，降低術語疲勞
- 禁止出現任何評分數字
