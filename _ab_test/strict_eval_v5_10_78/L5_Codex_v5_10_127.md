## 四、新發現工程 P0(Round 1 未抓、Round 2 浮現)

### P0-NEW-1 [code smell] `stripRawMarkdown` 從 1 個 catch-all 膨脹為 5-chain
- **檔案**:`page.tsx:768-779`
- **問題**:rename 後函式變成「marker stripper + 文案改寫器」hybrid、單一職責再次破壞:
  ```ts
  function stripRawMarkdown(html: string): string {
    return (html || '')
      .replace(/\*\*/g, '')
      .replace(/十五系統/g, '十四系統')   // ← 文案改寫、不是 strip markdown
      .replace(/十五套/g, '十四套')
      .replace(/15\s*系統/g, '14 系統')
      .replace(/15\s*套/g, '14 套')
  }
  ```
- **影響**:任何 prompt 改派(如未來 13 套)、frontend 又加一條 `.replace`、N+1 patch、最後 5→10→20 條
- **修法**:拆 `stripRawMarkdown` + `applyExternalSystemCount(html, count: 14)` 兩函式、後者集中所有「對外清零」rule、可被 prompt cascade 後撤掉

### P0-NEW-2 [hooks deps] CollapsibleSection useEffect deps 不一致
- **檔案**:`components/CollapsibleSection.tsx:54-65, 68-92`
- **問題**:第一個 useEffect dep `[expanded, isAnimating]`、第二個 `[expanded]` 但 setState `isAnimating`(L72/82/89);ESLint `react-hooks/exhaustive-deps` 會 warn、Strict Mode double-invoke 可能引入新競態
- **修法**:第二個 useEffect 內部用 `setIsAnimating` setter form、確保不依賴 closure;或加 deps 但測試 transition 不抖動

### P0-NEW-3 [test gap] v5.10.124 P0×P0 級 mobile bug 無 regression test
- 殭屍空白 = production 客戶看不到 6 件 mobile 報告全章節、屬災難級
- 修法 = 10 LOC、但 0 個 jest test 證 ResizeObserver 不再 lock contentHeight
- 下次 contributor 改 CollapsibleSection animation、極可能 break、無 CI 攔截
