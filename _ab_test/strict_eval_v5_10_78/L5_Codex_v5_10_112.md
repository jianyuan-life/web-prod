## 四、Top 5 工程級 P0(必修才可接近 95)

### P0-1 [security / XSS] sanitizeReportHtml 名實不符、攻擊面被靜默放大
- **檔案**:`app/report/[token]/page.tsx:763-765`
- **現況**:函式名 `sanitize` 但**無任何 sanitization**(早期 v5.3.43 註解說 passthrough、現只多一個 `replace(/\*\*/g, '')`)
- **風險**:
  - birth_data.name 雖在結帳時驗證、但無 HTML escape 保證;若上游驗證放寬 / customer_note 注入 `<img onerror>` → 全頁面 9 個 `dangerouslySetInnerHTML` 都會中
  - escapeHtml 只在 renderInlineMarkdown 內部跑、`personalityCard.rawContent` / hero / chip 多 component 走 sanitizeReportHtml 直接吐 HTML、未過 escapeHtml
- **修法**:改名 `stripRawMarkdown` 並引入 DOMPurify(已寫進待辦 #19、本 session 仍延期);或至少 strip `<script>` / `<iframe>` / `on*=` attribute

### P0-2 [anti-pattern] 治標暴力清取代治本、且根因尚未驗證
- **檔案**:`page.tsx:763-765`(v5.10.112)+ `page.tsx:856-859`(v5.10.105)+ `prompts/c_plan_v2.ts:932-936`(v5.10.109)+ `workflows/generate-report/plan-prompts.ts:21-29`(v5.10.110)
- **問題鏈**:
  1. AI 生不平衡 `**`(2-9% 漏率) → frontend `replace(/\*\*/g, '')` 暴力清(已知誤殺合法 bold)
  2. 同時加 prompt rule 要求 AI 平衡 — **但 ETHICS_RULES 與 c_plan_v2「建議、不是硬規格」段落同檔**(L932 緊接 v5.10.109 加的 markdown bold 規則)
  3. ETHICS_RULES 第 0 條雖標「絕對不可違反」、無 quality gate 自動 reject 不平衡 `**` 的報告
- **修法**:
  1. qualityGate 加 `if (countUnbalancedAsterisks(content) > 0) reject()`、強制 AI 重生
  2. 移除 frontend 暴力清、改用 marked / markdown-it 正式 parser(stack-based 平衡、不會誤殺合法 bold)
  3. 寫 unit test 覆蓋:平衡 `**X**`、不平衡 `**X`、跨行 `**X\nY**`、引用 `**《XXX》**`
