## 整合性檢查

### 1. 寬表識別準確度(20/20)
- L682 `colCountForLayout = (firstRow.match(/<td|<th/g) || []).length` 從 first row 計算 col 數、跟既有 sticky 路徑(L675)同來源 — **一致**
- 閾值 `>= 5` 切 Card Stack、`<= 4` 走 sticky;與 Codex/Gemini cross-source 共識(寬表才是 col-1 截斷主場)對齊 — **判定準**
- 短表(2-4 col、e.g. 五行屬性 / 簡易對照)維持 sticky 不破壞既有 UX — ✅

### 2. PartHighlights / PDF 影響(15/15)
- `PartHighlights.tsx` 不直接吃 markdown table、改動為 inline render 函式內、**無依賴 break**
- PDF 路徑 = `@media print` (L1877) override 全部背景白底、Card Stack 用 `rgba(255,255,255,0.02)` + `border:1px solid rgba(201,168,76,0.15)`;print 時 border 仍會印、視覺改變但**不破壞** — ✅(P2 改善:可加 `.report-card-stack { background: white !important; border: 1px solid #ddd !important; }` 進 @media print)
- `lib/report-structure.ts` extractTLDR / cleanFinalReport 為 string-level、不 parse table DOM — **無影響**

### 3. Mobile globals.css 衝突(13/15)
- L415-472 `.table-breakout` 整套 mobile cascade 把 thead/tbody/td 強制 flex column
- **Card Stack 路徑沒套 .table-breakout class**(L700 `class="report-card-stack"`)→ mobile cascade 不觸發、**好**(設計同 Card Stack、避免雙重 transform)
- **P1 finding**:Card Stack 自身在 mobile <375px 沒額外 query、`min-width:70px` label + `flex` row 在窄寬會推 value 換行;**建議**加 `@media (max-width:375px) .report-card-stack > div { flex-direction: column; gap: 4px; }` 防 iPhone SE / Galaxy 小屏壓縮

### 4. 對外清零(stripRawMarkdown)兼容(13/15)
- Card Stack `title`(L693)= cells[0] 已經 `.replace(/<[^>]+>/g,'')` strip HTML、但 **沒過 stripRawMarkdown 14↔15 cascade**
- AI 若在 cell 寫「15 套系統」→ Card title 會直接 render 成 stale 數字
- `cleanFinalReport`(string-level pre-render) 應已過、但 **未 cross-check**;**P1 建議**在 L693 `title` 前 chain `.replace(/15\s*套/g, '14 套')` 防漏

### 5. SEO / a11y(13/15)
- Card Stack 用 `<div>` 而非 `<dl>/<dt>/<dd>`、screen reader 無語義(把表格改 div = 失去 `role=table`)
- **P1 finding**:加 `role="table"` 到 outer + `role="row"` / `role="cell"` 到內層、或改用 `<dl>`(definition list 語義最近)
- title 用 `<div>` 而非 `<h4>` — a11y outline 跳過;**P2 建議**升 `<h4>` 但要 reset margin
- 無 `aria-label` 標明 card 是同一 row 的展開、**減 2 分**

### 6. PDF / print 兼容(13/15)
- `@media print` L1880 only target `.section-card / .glass / .report-bold` 等、**Card Stack 沒專屬 print rule**
- 印出來會是深色 `rgba(255,255,255,0.02)` 底 + 金字、low-contrast、可能客戶截 PDF 不可讀
- **P0 建議**(印 PDF 對鑑源核心交付):加 `@media print { .report-card-stack > div { background:white !important; border:1px solid #ccc !important; } .report-card-stack span { color:#333 !important; } }`

### 7. 防 regression(18/20)
- 不動 `<= 4` col 短表 sticky 路徑 — ✅ 既有 stable case 無 regression
- bodyTrList match `/<tr[^]*?<\/tr>/g`(L688)= greedy 但 lazy quantifier、跟既有風格一致 — 安全
- `cells.length === 0` early return `''`(L692)防 empty row 渲染空 card — ✅
- **P1 finding**:**未 unit test**;Card Stack 是純 string transform、可 jest 測 5/6/7 col + empty cells + special chars(< > & " ');無 test = regression risk
- **P1 finding**:`finalBodyRows` v5.10.156 加的 placeholder row(L676)若被 Card Stack render = 7 col 全 `—`、等於 noise;**建議** L688 抽 bodyTrList 後 filter 掉純 `—` row

## 判定:**HOLD(90/100、未達 ≥ 95 PASS 閾值)**

### 必修(才能 PASS)

**P0**(1 條):
- @media print 加 `.report-card-stack` rule(白底 + 黑字 + #ccc border、避免 PDF 不可讀) — 鑑源 PDF 是核心交付、不修等同於對寬表客戶 PDF degrade

**P1**(4 條):
- 加 `role="table"` / `role="row"` / `role="cell"` 或改 `<dl>` 結構(a11y 合規、screen reader 可讀)
- title cell 過 stripRawMarkdown 14↔15 cascade(防對外清零漏網)
- @media (max-width:375px) Card Stack 改 `flex-direction:column`(iPhone SE 防擠)
- jest unit test 覆蓋 5/6/7 col + empty/special chars(防未來 regression)

**P2**(2 條):
- title 從 `<div>` 升 `<h4>`(a11y outline)
- finalBodyRows placeholder row(L676)Card Stack render 前 filter 掉純 `—`

### 整體判斷

**設計方向正確** — 永絕 col-1 截斷的根本繞開、不再賭 sticky CSS hacks。架構乾淨、不破壞 ≤ 4 col 既有路徑、PartHighlights / cleanFinalReport / extractTLDR 全無 break。

**HOLD 真因** — 三件事漏:print PDF 兼容(P0)、a11y 語義(P1)、對外清零 cascade(P1)。**修這 3 條 + 加 1 條 mobile media query = 立刻 ≥ 95 PASS**。寬表問題 8+ 次糾正、本次架構是對的、但 production-grade 還差最後一哩。
