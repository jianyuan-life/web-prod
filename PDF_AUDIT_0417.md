### 1.2 字體問題（中度）

老闆規範：「字體 Microsoft JhengHei」。
實際：Fly.io Linux 容器只有 `WQY Zenhei`（文泉驛正黑），因為：
- Microsoft JhengHei 有商業授權，無法裝在 Linux 容器
- NotoSansCJK 是 CFF outline，ReportLab TTFont 不相容
- WQY Zenhei 是 TrueType，能正常渲染繁體中文

**判斷**：這是實際可用的最佳選擇，視覺上與 JhengHei 差異小，可接受。如需完全一致，需改用 HTML → Chromium 渲染的 PDF 方案（工作量大）。

### 1.4 空白頁問題（中度，僅 C_何宣逸Jamie 有）

| PDF | 空白頁 | 原因 |
|:---:|:---:|:---|
| C_boji.pdf | 僅 p2（目錄空白）| ⚠️ 目錄沒生成 |

空白頁本質：舊版 PDF 生成器（15 系統格式）在章節切換用 `PageBreak` 而非 `CondPageBreak`，若上一章結尾在頁底、下一章從新頁開始，中間會產生只有頁首頁尾的空白頁。

**修復狀態**：新版 11 章主題式格式（`pdf_routes.py:676-677`）已改用 `CondPageBreak(80)` 防範。舊版客戶 PDF 不會重生，是歷史遺留。

### 1.5 目錄（ToC）問題（輕度）

C_boji.pdf p2 提取到的目錄只有「目錄」兩個字，沒有列出章節。檢查原因：
- `pdf_routes.py:1219` `valid_titles = [... if t.strip()]` 過濾後若所有章節標題尚未記錄頁碼（page_map 空）→ 條目仍應顯示但頁碼為空
- 該 PDF 的目錄**實際可能有渲染**但 PyMuPDF text extraction 沒抓到（可能是畫布繪製而非文字流）

**建議**：實際打開 C_boji.pdf 目視確認。如果視覺上目錄確實空白，需檢查 `_build_toc_elements()` 的 `page_map` 傳遞。
