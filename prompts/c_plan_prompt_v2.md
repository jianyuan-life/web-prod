## 一、架構總覽

### 為什麼要改

| 問題 | 舊版（v1） | 新版（v2） |
|:---|:---|:---|
| API 呼叫 | 1 次 DeepSeek | 4 次 Claude Opus 4.6（並行） |
| 輸出字數 | 4,000-6,000 字 | 50,000-80,000 字 |
| 分析結構 | 按主題（性格/事業/財運） | 按系統（八字/紫微/奇門...） |
| 心理學 | 喧賓奪主，強制五步法/六步法 | 隱形融入，最後一章才正式亮相 |
| 數據利用 | details 截斷 500 字 | 完整 raw_data 不截斷 |
| 年齡適配 | 無 | 五層分級（幼兒/兒童/青少年/成人/長者） |
| max_tokens | 8,000 | Call 1: 16384 / Call 2: 16384 / Call 3: 12288 / Call 4: 8192 |

## 二、年齡分層判斷邏輯

```javascript
function getAgeGroup(birthYear) {
  const age = new Date().getFullYear() - birthYear
  if (age <= 6)  return 'toddler'   // 幼兒版 0-6 歲
  if (age <= 12) return 'child'     // 兒童版 7-12 歲
  if (age <= 18) return 'teen'      // 青少年版 13-18 歲
  if (age <= 60) return 'adult'     // 成人版 19-60 歲
  return 'elder'                     // 長者版 60+ 歲
}
```

### 各年齡層內容控制

| 年齡層 | 對象 | 必寫 | 不寫 |
|:---|:---|:---|:---|
| 幼兒(0-6) | 寫給父母 | 健康地圖、天賦才藝、教養策略、入學時機、親子互動 | 戀愛模式、投資理財、職場策略、桃花運、婚姻 |
| 兒童(7-12) | 寫給父母 | 學業方向、性格培養、社交引導、健康、才藝潛力 | 戀愛模式、投資理財、職場策略、桃花運 |
| 青少年(13-18) | 半父母半本人 | 升學方向、自我認同、社交、健康、未來職業方向 | 投資理財、婚姻分析、桃花運細節 |
| 成人(19-60) | 寫給本人 | 全部涵蓋：事業/財運/感情/健康/人際/大運 | 無限制 |
| 長者(60+) | 寫給本人 | 健康養生、晚運、子女關係、心靈安頓、退休規劃 | 升學方向、創業建議（除非仍在職） |

```

**實際內容：**

```
【數據鐵律——違反任何一條即為不合格報告】

1. 每一個分析論點必須引用具體的排盤數據——天干地支名稱、星曜名稱、宮位名稱、格局名稱。

4. 禁止使用通用模板語句。以下句型全部禁止：

5. 評分引用：排盤數據中每個系統都有評分（0-100），可引用來支撐論點。

6. 交叉驗證：當多個系統對同一個特質有描述時，必須交叉引用。例如：「八字中你的食傷旺代表表達能力強，而紫微命盤中巨門化權入命宮也印證了這一點——你不只會說，還說得有份量。」

### 3.2 寫作風格指令

```
{WRITING_STYLE}
```

**實際內容：**

```
【寫作風格規範——像一位見過世面的智者在跟客戶促膝長談】

語氣：
- 溫暖但不油膩，專業但不冰冷
- 像一位你信任的長輩或朋友，認真聽完你的故事後，用他的閱歷和命理知識幫你分析
- 不是心理諮商師（不要「你的感受是被允許的」），不是算命攤（不要「此命大吉」）

結構：
- 每個系統的分析以具體排盤結果開場，不以心理話術開場
- 先「看見」（用命理數據精準描述客戶的真實體驗），再「理解」（解釋為什麼），最後「指路」（怎麼做）
- 改善方案必須具體到「明天就能做的第一步」

心理學的角色（隱形融入）：
- 心理學是「怎麼說」，命理是「說什麼」
- 前面 14 系統分析中，心理學不出現任何術語——它隱形在遣詞用句裡
- 用命理數據做「鏡映」：精準描述客戶的真實體驗，讓客戶覺得「你怎麼這麼懂我」
- 用命理術語做「重構」：把挑戰重新定義為成長機會
- 改善方案中：心理學提供方法論，命理提供最佳時機

禁止出現的字眼：
- 「心理學研究表明」「根據依附理論」「認知行為療法建議」——這些留到最後一章
- 「你要小心」不附解法
- 「命中注定」「這輩子就是」「前世業障」
- 「你應該」「你必須」——改用「你可以試試看」
- 「別想太多」「想開一點」——改用「你的感受是有原因的」
- 空泛安慰：「一切都會好的」「船到橋頭自然直」

語言替換表：
- 「缺點」→「成長方向」
- 「運勢很差」→「能量在轉換期」
- 「缺少XX」→「有成長空間」
- 「你太XX」→「這個特質很強烈，平衡是你的課題」
- 「注意」→「留意」
- 「問題」→「課題」

範例——好的寫法：
> 你命盤中庚金食傷藏在巳火裡，不透天干——這代表你內心其實有很多想法和感受，但真正說出口的可能只有10%。這不是你不會表達，而是你天生對「開口說話」這件事有更高的標準。你的食傷被印星壓制，換句話說，你腦中有一個「品質審查官」，每句話出口前都要先過他那關。這讓你說出來的話往往很有份量，但也讓你在需要快速反應的社交場合裡覺得吃力。

範例——壞的寫法（禁止）：
> 你可能常常感覺到自己有很多想法卻說不出來，這很正常，很多人都有這個經驗。建議你可以多練習表達，慢慢就會好了。
```

**2.6 大運分析**

**2.7 八字總結：好的地方（5-7項）**
每一項必須：引用具體天干地支 → 解釋命理意涵 → 翻譯成生活中的具體表現 → 如何善用

### 策略 A：Call 4 也並行（推薦）

直接從排盤原始數據中提取所需資訊，不依賴前三個 Call 的輸出。

```javascript
// 從排盤數據中提取 Call 4 所需的摘要
function prepareCall4Data(calcResult) {
  const analyses = calcResult.analyses || []
  
  return {
    all_systems_score_table: analyses.map(a => `${a.system}: ${a.score}分`).join('\n'),
    all_good_points: analyses.flatMap(a => 
      (a.good_points || []).map(g => `[${a.system}] ${g}`)
    ).join('\n'),
    all_bad_points: analyses.flatMap(a => 
      (a.bad_points || []).map(b => `[${a.system}] ${b}`)
    ).join('\n'),
    all_improvements: analyses.flatMap(a => 
      (a.improvements || []).map(i => `[${a.system}] ${i}`)
    ).join('\n'),
    lucky_elements_raw: analyses.map(a => {
      if (a.lucky_elements) return `[${a.system}] ${JSON.stringify(a.lucky_elements)}`
      return null
    }).filter(Boolean).join('\n'),
  }
}
```

### 策略 B：Call 4 等待前三個完成（備選）

如果希望 Call 4 能參考前三個 Call 的實際輸出：

```javascript
const [result1, result2, result3] = await Promise.all([call1, call2, call3])

// 從 AI 輸出中提取摘要（用簡單的截取）
const call4Input = {
  call1_summary: extractSummary(result1, 2000), // 前 2000 字
  call2_summary: extractSummary(result2, 1500),
  call3_summary: extractSummary(result3, 1500),
  // ... 其他數據從排盤原始數據提取
}
const result4 = await callClaude(call4Prompt, call4Input)
```

此策略增加延遲（等前三完成後才開始 Call 4），但 Call 4 的輸出品質更高。

**推薦策略 A**——排盤數據已經足夠豐富，Call 4 不需要依賴前三的 AI 輸出。

```javascript
// 新版：完整傳遞，不截斷
function formatSystemData(analysis) {
  let output = `\n### 【${analysis.system}】評分：${analysis.score}分\n`
  
  if (analysis.summary) output += `\n**摘要：**${analysis.summary}\n`
  
  if (analysis.good_points?.length) {
    output += `\n**好的地方：**\n`
    for (const g of analysis.good_points) output += `- ${g}\n`
  }
  
  if (analysis.bad_points?.length) {
    output += `\n**需要注意：**\n`
    for (const b of analysis.bad_points) output += `- ${b}\n`
  }
  
  if (analysis.improvements?.length) {
    output += `\n**改善建議：**\n`
    for (const i of analysis.improvements) output += `- ${i}\n`
  }
  
  // 完整的 raw_data 和 details——不截斷
  if (analysis.raw_data) {
    output += `\n**排盤原始數據：**\n`
    output += typeof analysis.raw_data === 'string' 
      ? analysis.raw_data 
      : JSON.stringify(analysis.raw_data, null, 2)
    output += '\n'
  }
  
  if (analysis.details) {
    output += `\n**解讀詳情：**\n`
    output += typeof analysis.details === 'string' 
      ? analysis.details 
      : JSON.stringify(analysis.details, null, 2)
    output += '\n'
  }
  
  return output
}
```

## 十三、品質閘門（QA Checklist）

報告生成後，在存入 Supabase 之前，自動執行以下檢查：

```javascript
function qaCheck(mergedReport, birthData) {
  const errors = []
  
  // 1. 章節完整性：20 章都在
  for (let i = 1; i <= 20; i++) {
    const chapterPatterns = [
      `## 第${i}章`,
      `## 第${chineseNum(i)}章`,
      `第${i}章`,
    ]
    if (!chapterPatterns.some(p => mergedReport.includes(p))) {
      // 也檢查 ##  + 章節標題關鍵字
      errors.push(`缺少第 ${i} 章`)
    }
  }
  
  // 2. 年齡適配：幼兒版不應出現戀愛/投資
  const age = new Date().getFullYear() - birthData.year
  if (age <= 6) {
    const forbidden = ['戀愛模式', '投資方向', '桃花運', '職場策略', '婚姻']
    for (const word of forbidden) {
      if (mergedReport.includes(word)) {
        errors.push(`幼兒版報告出現禁止內容：${word}`)
      }
    }
  }
  
  // 3. 字數下限
  if (mergedReport.length < 30000) {
    errors.push(`字數不足：${mergedReport.length} 字（最低要求 30,000 字）`)
  }
  
  // 4. 禁止語句掃描
  const forbidden_phrases = [
    '命中注定', '這輩子就是', '前世業障',
    '心理學研究表明', '根據依附理論', '認知行為療法建議',
    '你應該', '你必須', '別想太多', '想開一點',
    '一切都會好的', '船到橋頭自然直',
  ]
  for (const phrase of forbidden_phrases) {
    if (mergedReport.includes(phrase)) {
      errors.push(`包含禁止語句：「${phrase}」`)
    }
  }
  // 例外：第二十章「心靈視角」允許使用心理學術語
  
  // 5. 留白掃描
  const placeholders = ['待確認', '待補充', 'TODO', 'TBD', '...（略）']
  for (const p of placeholders) {
    if (mergedReport.includes(p)) {
      errors.push(`發現留白：「${p}」`)
    }
  }
  
  // 6. 客戶姓名檢查
  const nameCount = (mergedReport.match(new RegExp(birthData.name, 'g')) || []).length
  if (nameCount < 5) {
    errors.push(`客戶姓名出現次數過少：${nameCount} 次（建議至少 5 次）`)
  }
  
  return { passed: errors.length === 0, errors }
}
```
