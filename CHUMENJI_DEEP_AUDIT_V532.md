## 二、Workflow 與 Prompt 層

### 2.1 流程完整性
```
index.ts → callChumenjiTop(planCode, birthData)
        → chumenjiTop 注入 aiGenerateGeneric
        → buildGenericUserPrompt 強制注入「引擎計算結果 — 不可更改」
        → Claude Opus 4.6 生成 → validateReportAgainstData → qualityGate
```
流程**架構正確**，每一步都有走。

### 2.2 【最大漏洞】品質閘門沒比對 AI JSON 與引擎結果
**原始 qualityGate 只檢查**：
- 是否有 === TOP3_JSON_START === 區塊
- 是否有 9 個章節關鍵字
- 字數 > 2500

**缺失**：AI 寫出來的 `{"date":"2026-05-13","direction":"西北","title":"生門+天輔+六合"}` 是否**真的等於**引擎輸出？

**本次修復（已寫入 `workflows/generate-report/steps.ts`）**：
```typescript
if ((planCode === 'E1' || planCode === 'E2') && chumenjiTop?.results?.length) {
  // 抽 AI JSON → 逐筆比對 engineList
  // date/time_start 必須匹配；不匹配 push 硬門檻 warning
  // direction 不符也打硬門檻
}
```
歸入 `[硬門檻]` 前綴 → 進入 hardFailures → 觸發 retry（最多 3 次）→ 3 次仍失敗 `markReportFailed`。

### 2.3 Prompt 強化（本次改）
加入 7 條「違反視為廢稿重寫」規則（原本 4 條），並**明確禁止 AI 自行捏造門星神組合或評分**。
