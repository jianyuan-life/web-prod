**問題**：正常紫微盤 14 主星分佈 12 宮，空宮數通常是 4-5 個。**11 空宮是不可能的，代表除了命宮以外所有星曜都沒安上去**。

**推測根因**：五行局計算失敗（下一條 bug），導致紫微星起的位置錯誤，後續 13 星全部沒安進去。

### 3.3 🔴P0：五行局顯示「局」（資料綁定 bug）

- 何宣逸頁面：「命宮主星：天府（令星）· 庚年生 · 局」
- 林沅霖頁面：「五行局：局」
- **應顯示**：火六局 / 水二局 / 木三局 / 金四局 / 土五局

**根因**：前端模板用「{五行局}局」組合，但後端 `wuxing_ju` 欄位為空字串，只剩「局」字。

這是上一條「11 空宮」的直接原因——沒五行局就算不出紫微星起盤位置。

### 3.4 🔴P0：林沅霖閏月處理錯誤

- 西曆 1993-05-20 = 農曆癸酉年閏三月廿九
- 頁面結果：命宮在**子宮**，主星**貪狼**（忌）
- 老闆預期：命宮在**丑宮**，主星**天同+巨門**
- 差異：閏三月廿九屬於「後半閏當下月」（四月）還是「前半閏當本月」（三月）？
- 紫微標準慣例：閏月**前 15 天當本月，後 15 天當下月**
- 閏三月廿九 → 第 29 天 → 應當**四月**處理
- 四月時：寅起正月 → 卯(二) → 辰(三) → 巳(四)，辰時從巳宮逆數 4 格 = 丑宮 ✓

**結論**：系統把閏月全當三月處理，沒做「前半本月、後半下月」拆分 → 命宮差一宮 → 整個命盤結構錯。

### 3.5 🔴P0：AI 深度解讀大量簡體中文

何宣逸頁面的「命盤深度解讀」段落：
- 「稳重」「财富」「难以」「关系」「风险」... 全是簡體
- 頁面 title 宣告繁體，但解讀段落是簡體 → 違背品牌一致性

林沅霖頁面也一樣。推測 AI prompt 沒指定 `zh-TW` 或未做 opencc 後處理。

### 3.6 🟡P1：AI 解讀 Markdown 粗體殘留

林沅霖 AI 解讀：
- 「1. **個性開朗**：貪狼星坐命的人...」
- 「2. **愛好廣泛**：...」
- 「3. **富有魅力**：...」

應渲染為實際粗體，不應留 `**` 在文字中。

## 6. 跨工具共通問題

### 6.1 🟡P1：CSP 白名單缺 googletagmanager.com

Console 錯誤：
```
Connecting to 'https://www.googletagmanager.com/td?...' violates CSP directive:
"connect-src 'self' https://*.supabase.co https://api.stripe.com
 https://*.fly.dev https://www.google-analytics.com ..."
```

**影響**：GA4 的後端上報（td API）被阻擋，可能造成分析數據缺失。

**修法**：把 `https://www.googletagmanager.com` 加到 CSP `connect-src` 白名單。

### 6.2 🟡P1：AI 解讀段落簡繁混用
- 紫微：AI 解讀整段簡體（「稳重」「财富」「难以」）
- 姓名學：AI 解讀繁體但 Markdown 殘留（`**人格數16**`）
- 八字：AI 解讀繁體，但納音表欄位簡體（「钗钏金」「剑锋金」）

**統一修法**：加 opencc `s2twp` 後處理 + Markdown `**` 清理。

### 6.3 🔴P0：手機版 RWD 水平溢出
- 八字 tools 頁 390px 視窗下 scrollWidth=317，viewport=312
- 右側有橫向滾動條，輸入欄位、標題被截斷

**修法**：檢查所有絕對寬度或 min-width 超過 viewport 的元素，改 `max-w-full` + `overflow-x-hidden`。

### 關鍵判斷
1. **紫微是最嚴重的**：五行局計算失敗導致連鎖效應（11 空宮），整個命盤邏輯崩壞
2. **奇門是資料綁定斷層**：時柱正常但年月日柱顯示 `-`，看起來是前後端介接問題
3. **八字計算核心 OK，但邊緣計算有 bug**：旬空、流年偏移、納音簡繁
4. **姓名學是唯一接近商用品質的**：核心公式全對，只要清掉 Markdown 就能上

### 給老闆的建議
- **緊急修 P0 的 9 項**後，要求開發部門做完整 regression
- 紫微和奇門修完後，用至少 5 個已知命盤的客戶資料做端對端回歸（何宣逸、林沅霖、再加 3 人）
- 建議加入自動化測試：每次部署後用固定的 5 個客戶跑驗證，和預期值比對
- 目前不建議把這 4 個工具放在首頁宣傳「免費排盤」，因為錯誤率高會損害品牌
