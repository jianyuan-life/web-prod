## 二、Layer 1 — 命理合盤算法審閱

### 程式碼結構
```

```

### 現有流程（正確）
1. 從已完成的人生藍圖報告載入 2-8 位家庭成員
2. `extractKeyDataForFamily()` 只抽每份個人報告的「關鍵命理標記」（不餵全文，避免 AI 重複個人分析）
3. 餵給 Claude Opus 4.6，用 128K context 做家族互動分析
4. 品質閘門檢查 10 個必要章節 + 字數下限（2 人 8K / 3 人 10K / 4 人+ 12K 字）
5. AI 審核員評分（< 70 告警）
6. 生成 PDF，上傳 Supabase Storage

### 結論
**結構正確、無明顯致命 bug。** 問題出在 Prompt 對「論證嚴謹度」的約束不夠強——LLM 在高溫度寫作時會堆砌多系統、循環論證、給出執行悖論的建議，需要在 Prompt 層強化防禦。

### 各 LLM 主要批評重點

#### DeepSeek-R1（邏輯推演，最嚴格）
> 「報告混雜八字、紫微、人類圖、西洋占星、吠陀、塔羅等多體系術語，未建立任何體系間的邏輯橋樑或轉換規則。核心論斷（如月冥對分讓她成為情緒雷達）屬直接斷言，存在『因為命格 A 所以行為 B』的論證跳躍。」

> 「核心建議存在執行矛盾：『家庭暫停機制』要求情緒上頭的成員理性發起暫停，這與報告所述暴衝特質（情緒洪水時前額葉下線）存在執行悖論。」

> 「分析存在循環論證：用命理配置解釋行為，又用同一行為反證命理有效。對 2026 年運勢的斷言『全家特別累特別燥』近乎宿命論，未考慮變量與干預可能。」

#### GPT-4o（家庭心理學）
> 「命理分析過於籠統，缺乏具體的理論支持。建議雖多但缺乏具體可操作性。敘事冗長，重複性高，缺乏一針見血。」

#### Qwen-Max（中文命理）
> 「對特定宮位和星曜的解釋缺乏深入專業論述。部分段落偏向個人分析未展現互動動態。」

#### Kimi-128k（敘事）
> 「部分段落冗長、專業術語無解釋、缺年齡分層指導。」

## 四、Layer 3 — 歸納的系統性問題與根因

**三個 LLM（DeepSeek/GPT-4o/Qwen）都扣分在同一批問題上，證明是 Prompt 層面的系統性漏洞，不是 AI 當天狀態。**

| # | 問題 | 根因 | 嚴重度 |
|:---:|:---|:---|:---:|
| 1 | 多系統論證無橋樑 | Prompt 只要求「交叉引用」沒要求「先主後輔、明確銜接」 | P0 |
| 2 | 循環論證（命格→行為→回頭證命格）| Prompt 沒明禁此模式 | P0 |
| 3 | 建議執行悖論（觸發條件下當事人無法執行）| Prompt 沒要求建議有「備援」或「第三方觸發」| P0 |
| 4 | 絕對化流年斷言 | Prompt 鼓勵「一針見血」未限制確定性詞彙 | P1 |
| 5 | 自相矛盾（同現象既是優勢又是問題）| Prompt 沒要求「區分情境」 | P1 |
| 6 | 部分段落泛泛而談 | Prompt 已有「必涉兩人」但無硬性驗證機制 | P1 |

## 五、Layer 4 — 已執行的修正

### 修正 1：Prompt 層面新增「論證嚴謹性鐵律」（P0 + P1 全覆蓋）
**檔案：** `workflows/generate-report/plan-prompts.ts`（G15 區塊）
**位置：** 數據規範後新增六條鐵律

1. **多系統論證必須有橋樑** — 禁止三個以上系統平鋪堆砌，必須明確「主論據/輔助驗證」
2. **禁止循環論證** — 命格→行為傾向→日常場景（單向），不可反向引用
3. **禁止執行悖論** — 每條建議必答「誰發起、觸發條件下對方能否執行、備援是什麼」
4. **禁止絕對化流年斷言** — 必須標「可能/傾向/需要觀察」，不得用「全家特別累/特別燥」這類一刀切
5. **禁止自相矛盾** — 同一特質若既是優勢又是課題，必須區分「X 情境下是優勢」「Y 情境下是風險」
6. **建議閉環檢核（四問）** — Q1 誰發起、Q2 觸發條件、Q3 對方如何配合、Q4 對方無法執行時的備援

### 修正 2：UI 新增「家族動力全貌圖」面板
**新檔案：** `components/FamilyDynamicsPanel.tsx`
**掛載點：** `app/report/[token]/page.tsx`（header 區塊結束、正文章節開始之前）

**三個視覺化區塊：**

1. **成員關係網（SVG 圓形布局）**
   - 每位成員為圓形節點（首字大字 + 五行標註）
   - 兩兩連線：共鳴（金色實線）/ 張力（橘紅色實線）/ 中性（灰虛線）
   - 自動從 AI 報告文字抽取關係類型（關鍵詞計分法）

2. **全家五行分佈**
   - 金木水火土 5 條水平條圖
   - 從 AI 報告抽取「{姓名} 日主 {天干}」自動對應五行
   - 「金水」「木火」特殊組合各計 0.5 人到兩種

3. **家庭角色矩陣**
   - 每位成員一張卡，顯示其角色標籤：決策者 / 協調者 / 執行者 / 情緒穩定器
   - 角色從 AI 報告關鍵詞自動抽取（如「決策者：XX」或「XX ... 穩定器」）
   - 一人可同時有多個角色

**設計原則：**
- 純 SVG + Tailwind 無第三方依賴
- 只對 G15 啟用（`isFamily && member_names.length >= 2`）
- 資料抽取失敗時優雅降級（顯示提示「報告內無明確標註，請參考完整章節」）

### 修正 3：PDF 生成無需改動
**理由：** PDF 引擎共用，呼叫時已傳入家族成員姓名合集作為 client_name，Prompt 修正生效後重生的 PDF 自動升級內容。PDF 版面本身不需要視覺化圖表（客戶以 web 版為主）。

## 六、驗證計畫（生成實測）

### 階段 A：小樣本 AI 重生（預計 1-2 份）
1. 取現有 4 人家族 birth_data
2. 用新 Prompt 重生一份 G15 報告
3. 跑 5 LLM 評分（使用 `g15_audit/run_5llm_review.py`）

**通過標準：** min ≥ 95 且 DeepSeek-R1（最嚴苛視角）≥ 95

### 階段 B：若階段 A 有 LLM < 95
對應升級規則：
- DeepSeek 指出「論證跳躍」→ 強化鐵律 1 的範例數量
- DeepSeek 指出「執行悖論」→ 強化鐵律 3 的 Q1-Q4 模板
- GPT-4o 指出「冗長」→ Prompt 加入「每段不超過 N 字」
- Kimi 指出「術語未解釋」→ Prompt 加入「首次出現必括號白話」

### 階段 C：批次生成 5 份確保穩定
不同家庭組合（2 人/3 人/4 人/5 人/夫妻+孩子），每份都 ≥ 95 才算通過。
